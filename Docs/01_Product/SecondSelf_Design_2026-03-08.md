## 1. 核心理念

### 「记录-进化」

Life Narrator 的核心不是「记录-回顾」，而是「记录-进化」。

区别在于：
- **记录-回顾**：日记 → 回看历史，被动
- **记录-进化**：日记 → 自我模型持续更新 → 认知差距分析 → 主动成为更好的自己

第二自我模块是「记录-进化」理念的核心载体。它不是一个聊天机器人，而是一个由用户自己的记录喂养、持续进化的「另一个自己」。

### 两个核心使用场景

1. **主题回顾**：「把我有关运动的所有体会整理一遍」
   → 检索相关 Atoms，以时间线或晶体卡片形式呈现

2. **第二自我模拟**：「我最近在考虑是否换工作，你觉得呢？」
   → AI 以用户自身的思维方式和价值观回应，不是给建议，而是「你会怎么想」

## 2. 架构决策记录

### 决策 A：自我模型存储方式

**选择**：四层结构（锚点 + 主体 + 当期补丁 + 忽略条目）+ 季度整合机制

**理由**：
- 覆盖模式会丢失历史，与「记录-进化」理念矛盾
- 纯补丁追加会导致忽略链越来越复杂，难以维护
- 季度整合（ConsolidationJob）将当期补丁归并为干净的主体，忽略条目同步整理，复杂度每季度归零

**忽略条目的作用**：不是「跳过某条补丁」，而是将用户所有忽略请求集中成一份负向指令清单，注入 Prompt，防止 AI 从新数据中重新发现并呈现用户已明确不想看到的内容。

### 决策 B：Proposal 存储

**选择**：独立 ProposalEntity（收件箱模式）

**理由**：周更新和月更新可能同时积压提案，用户应能批量审批而不丢失任何提案。

### 决策 C：第二自我的「声音」来源

**选择**：用户锚点 + AI 补丁叠加

- 用户手写一段基础「自我描述」作为永久锚点，AI 不可修改
- AI 通过周期 Job 持续追加补丁，细化和扩展锚点
- 冷启动体验好（锚点立即可用），长期越来越准（补丁不断丰富）

### 决策 D：补丁格式

**选择**：结构化自然语言

```
[2026-03-08 行为层补丁]
观察到：过去四周运动相关记录共 12 条，坚持晨跑的描述集中在周二至周四。
新增认知：运动习惯在工作日中段最稳定，周末反而容易中断。
证据引用：Atom#334, Atom#389, Atom#401
```

**理由**：既人类可读（用户审批时能理解），又可直接注入 LLM 上下文（无需格式转换）。随着上下文窗口增长，全量补丁可直接注入而无需压缩中间层。

## 3. 六层经验积累体系

```
Layer 1  原始 Capture        用户的流水账输入，完整保留
Layer 2  结构化 Atoms        语义最小单元，含来源偏移量
Layer 3  标签索引            隐性标签（置信度）+ 显性标签（用户）
Layer 4  晶体卡片            主题回顾的结构化输出单元
Layer 5  自我模型（活文档）  用户锚点 + 所有 active 补丁
Layer 6  进化追踪            补丁时间线，可回溯任意时间点的自我认知
```

越往上越精炼，越往下越原始。检索时按需选择粒度：精确场景用 Layer 3–4，自我认知场景用 Layer 5–6。

## 4. 周期更新 Job 设计

### 三种触发

| Job | 频率 | 分析窗口 | 输出 |
|-----|-----|---------|-----|
| BehavioralUpdateJob | 每周一 | 最近 7 天 Atoms | 行为/沟通偏好层补丁建议 |
| IdentityUpdateJob | 每月 1 日 | 最近 30 天 Atoms | 价值观/思维模式层补丁建议 |
| ContradictionDetector | 每次 Atom 写入后 | 新 Atom vs 当前模型 | 矛盾事件标记，累计 3 次触发小型 Job |

### 三个分析信号

1. **语言模式信号**：高频动词/形容词 → 沟通偏好和行为倾向
2. **重复主题信号**：跨时间段反复出现的关注点 → 当前核心上下文
3. **否定信号**：「不想」「放弃」「以前认为」→ 认知边界和价值观更新

### 铁律：AI 是「证据整理者」，不是「评判者」

- 只提 Proposal，不自动写入
- Proposal 内容描述观察到的事实，不做价值判断
- 用户有完整的接受 / 修改 / 忽略权利

## 5. 分层标签流水线

### 背景

标签库随时间增长（从几十到几百），全量注入会增加 token 成本并降低模型专注度。

### 两步设计

```
Step 1（轻量模型，< 1s）
  输入：Capture 文本 + 本地领域分类树（tag-taxonomy.json）
  输出：命中领域（1–3个）+ 粗粒度标签（2–5个）

Step 2（重模型，2–4s）
  输入：Capture 文本 + Step 1 筛选出的领域标签（~60个）
  输出：Atoms（含偏移量）+ 隐性标签（含置信度）+ 显性标签建议
```

### 成本对比（20条/天，每月）

| 方案 | 月费用 | 备注 |
|-----|-------|-----|
| 当前单步 GPT-4o | ~$4.86 | 标签占成本约 31% |
| 分层 GPT-4o-mini + GPT-4o | ~$3.86 | 推荐默认配置 |
| 全 DeepSeek V3（经济模式）| ~$0.34 | 用户知情选择，数据路由至中国 |

## 8. 关键架构洞察（来自设计讨论）

**四层结构面向未来**：季度整合控制了主体和补丁的总体积；忽略条目独立存储确保负向约束不随季度清空而丢失；当前上下文窗口已达数十万 token，全量注入四层内容已可行，无需额外压缩。

**提炼观察，不做评判**：系统从用户的行为记录和所思所想中提炼模式；「发现规律」是任务边界，不判断行为好坏。

**原始 Capture 不需要全量注入**：任何使用场景（主题回顾、第二自我模拟、周期 Job）都通过检索或时间窗口取子集，不需要喂入一年的原始记录。流水线的逐层蒸馏（Capture → Atoms → Tags → Patches）确保了信息密度的递增和体积的递减。
