# 2026-05-02 Session 006

## Goal
Prepare the app and backend so App Store/Release builds can use a public AI backend, and add an in-app feedback path that can carry a short description, contact method, and optional screenshot.

## Context
The App Store version showed `AI 配置缺失，无法拆分。` because Release builds do not receive local debug environment variables or debug-only API keys. The app needs a bundled/public backend URL configuration path, friendlier user-facing AI failure copy, and a feedback route so users can report issues without leaving the app.

## Implementation
- Added Release-safe AI backend URL lookup through `LifeNarratorAIBaseURL` and bundled `AppConfig.plist` `AIBaseURL`, while preserving the existing debug environment variable path.
- Added a shared app runtime user identifier so backend AI requests and feedback requests use the same stable local user id.
- Added `FeedbackScreen`, reachable from Settings and AI split failure states, with message, optional contact, and optional screenshot upload.
- Replaced user-facing missing AI configuration copy with service-unavailable wording so public users are not exposed to internal configuration language.
- Added backend `POST /v1/feedback` and `/admin/feedback`, JSONL storage, screenshot support, rate limiting, and admin dashboard links.
- Added server deployment scaffolding with `package.json`, `Dockerfile`, `.env.example` entries, and README deployment instructions.
- Updated the public support page to direct users toward in-app feedback.
