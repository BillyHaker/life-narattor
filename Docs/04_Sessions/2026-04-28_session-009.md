# 2026-04-28 Session 009 — Record all scope scrolls to latest

## Goal
When the Record page range filter is switched to `全部`, the list should land on the newest record instead of showing the oldest record at the top.

## Files Changed
- `Life Narattor/Screens/RecordFeedScreen.swift`
- `Docs/VERIFICATION_BACKLOG.md`
- `Docs/04_Sessions/2026-04-28_session-009.md`

## Implementation Notes
- Added a `selectedScope` observer inside the `ScrollViewReader` scope so range changes directly request latest-item scrolling with access to the active `ScrollViewProxy`.
- Kept the existing record ordering unchanged: sections and items still render oldest to newest.
- Delayed `proxy.scrollTo` by two main-queue turns so SwiftUI has time to apply the new filtered content before attempting to scroll to the latest item.
- Left search, filtering, capture creation, and persistence behavior unchanged.
