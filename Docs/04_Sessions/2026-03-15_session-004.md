# Session Log

## Goal
按“生活记录单元建模重构方案”落一版可运行实现：拆分链路由单步 atomize 过渡为 `clean -> semantic_chunks -> record_units -> tag_mapping`，并让拆分页直接展示结构化 record units。

## Plan
1. 扩展 atomize 输出结构，加入 `semantic_chunks`、`context_attributes`、`behavioral_chain`、`tag_hints`。
2. 更新 app / backend 的 atomize prompt 与 schema。
3. 将结构化 atomize payload 落到 `ArtifactEntity`，避免 UI 再次被旧 atom 文本压扁。
4. 调整拆分页优先读取结构化 payload 展示。
5. 清理重转写/重整理/重新拆分时的旧 atomization payload。
6. 验证 backend 语法和 Xcode build。

## Work Log
- 在 `AtomizationModels.swift` 中新增：
  - `SemanticChunkDraft`
  - `RecordUnitDraft.contextAttributes`
  - `RecordUnitDraft.behavioralChain`
  - `AtomizationArtifactPayload`

- `CaptureFeedViewModel` 在重转写、重新整理、重新拆分、revision 等重置路径上同步清理旧 `atomization_payload`，避免读到脏数据。
