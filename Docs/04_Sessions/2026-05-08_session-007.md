## Goal
Move the widened root tab bar closer to the bottom safe area and make it slightly slimmer, while keeping the three-layer Record bottom UI.

## Plan
1. Reduce Record bottom inset avoidance height so the root tab no longer looks excessively floating.
2. Reduce root tab item height and internal padding slightly.
3. Reduce root tab bottom padding to sit closer to the Home Indicator.
4. Verify build and document manual checks.
5. Commit one small visual refinement.

## Detection Plan
- Expected behavior: the root tab bar stays wide but sits closer to the bottom, with a slimmer capsule.
- Pass criteria: three bottom layers remain visible; root tab does not overlap the input row; root tab no longer appears to float too high; tap targets remain comfortable.
- Failure signals: input row becomes covered again, root tab becomes too small, or the Home Indicator area feels cramped.
- Regression surface: Record/Assistant switch, text input, voice recording chip, root tab switching, keyboard state.
