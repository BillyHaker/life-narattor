# Session 003 — AI-Native Memory UX Pass

## Goal
Rework the main iOS surfaces so Life Narrator feels like a lightweight memory container: low-pressure capture, quiet organization, and reviewable signals that surface when needed.

## Plan
- Record page: reduce management feel, emphasize quick capture and gentle status.
- AI Review page: shift from AI search to memory review prompts and traceable lines.
- Timeline page: make day cards feel like life playback, with softer narrative CTAs.
- Projects page: shift language toward durable `线索` instead of empty project folders.
- Verify with text scans, build, tests, and simulator screenshot inspection.

## Detection Plan
- Expected behavior: first screens use `记录 / 整理 / 回看 / 线索` language instead of heavy search/generate/manage language.
- Detection path: run text scans for old high-pressure terms, build with Xcode, test on simulator, inspect Record screenshot.
- Pass criteria: app builds, tests pass, navigation remains intact, key labels are updated, and empty states feel low pressure.
- Failure signals: build errors, broken navigation, missing input bar, old labels dominating first screens.
- Regression surface: record input, voice capture, AI review retrieval, timeline day navigation, project detail navigation.

## Work Log
- Read product north star and UI skills for capture, timeline, review, project, and assist archive constraints.
- Updated Record page header, search placeholder, empty state, input placeholder, and capture status copy.
- Updated AI Review entry copy, filter naming, starter prompts, loading text, empty/error text, and evidence section accent.
- Updated Timeline empty state and day card CTA language.
- Updated Day Detail AI response copy and narrative regeneration wording.
- Renamed visible Project tab/page language to `线索` while keeping existing data model unchanged.
- Updated weekly/monthly/tag review entry copy.
- Fixed voice completed status from `记录成功` to `已接住` after simulator screenshot inspection.
