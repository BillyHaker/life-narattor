# Session Log

## Goal
收紧 `record_units` 的组装规则，避免感受/结果被独立拆成新事项，同时过滤掉把主事件复写进 `result_or_state` 的明显错误结果。

## Plan
1. 在本地模型层增加 `record_units` 的归一化与合并逻辑。
2. 将纯状态/情绪型 unit 默认并入上一条主事项。
3. 过滤 `result_or_state` 中对主事项的重复复述，仅保留真正的结果、后果、感受与状态变化。
4. 同步收紧 atomize prompt，明确“一条 summary 只表达一个主事项，感受默认附着”。
5. 验证 server 语法与 Xcode build。

## Work Log
