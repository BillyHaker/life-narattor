# 2026-05-11 Session 001 - Onboarding Navigation and Parallel Entry Copy

## Goal
Fix the product onboarding guide so the Continue button reliably advances pages, and revise copy so direct recording and assistant-assisted recording are presented as parallel choices rather than ordered steps.

## Plan
1. Diagnose the current onboarding implementation and copy.
2. Replace TabView-driven page switching with explicit page rendering controlled by `pageIndex`.
3. Add lightweight drag gesture support for horizontal page changes.
4. Rewrite page eyebrows and header copy to remove sequential `第一步 / 第二步 / 第三步` framing.
5. Verify build/preview where possible.
6. Create Change Log and commit one focused change.

## Non-Expected Behavior Diagnosis

**Phenomenon:** User tapped `继续` on the onboarding guide, but the visible page did not advance. The guide also describes direct recording and assistant recording as `第一步` / `第二步`, implying a sequence that does not match the product model.

**Reproduction Steps:** Open product guide, stay on the first page, tap `继续`.

**Layer 1 - Implementation Check:** `OnboardingGuideScreen` uses `TabView(selection:)` with `.page(indexDisplayMode: .never)` and mutates `pageIndex` from an external footer button. This depends on PageTabViewStyle synchronizing its selection with an outside state mutation. On the reported build this appears unreliable.

**Layer 2 - Spec Check:** The onboarding decision docs frame onboarding as a lightweight guide. The current copy over-specifies a sequence for two parallel entry modes.

**Layer 3 - Product Check:** The product direction is correct: users should either directly record or use the assistant, depending on readiness. The bug is implementation stability plus copy framing, not the underlying feature.

**Root Cause Judgment:** Implementation-level navigation fragility plus UX copy mismatch.

**Suggested Fix:** Render only the current page from `pages[pageIndex]`, drive all page changes directly from local state, and update page labels to named modes instead of steps.
