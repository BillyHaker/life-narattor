# Session Log

## Goal
把“显性标签只来自用户主动输入或用户接受推荐”这条边界落实到当前标签写入链，避免 AI 推荐阶段直接创建显性标签。

## Plan
1. 收紧推荐标签写入路径，禁止推荐阶段直接创建显性标签。
2. 让新标签候选先以隐藏标签 + 建议链接的形式存在。
3. 用户确认建议标签时，再将对应标签升级为显性标签。
4. 验证 Xcode build。

## Work Log
- 调整 `AtomTagStore.swift`：
  - 推荐显性标签只允许复用已有显性标签；
  - 不再在推荐阶段通过 `addTag(...)` 创建新显性标签。
- 新标签候选现在会先通过 `ensureHiddenTag(...)` 建成隐藏标签，并以 `isSuggested = true` 形式挂到 atom。
- `fetchTagMap(...)` 放宽为：
  - 显性标签始终可见；
  - 隐藏标签只有在 `isSuggested = true` 时才会显示给用户作为候选。
- `confirmSuggestedTag(...)` 现在会在用户确认时把隐藏标签升级成显性标签，并同时把建议关系转成正式关系。
