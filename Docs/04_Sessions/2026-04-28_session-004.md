# 2026-04-28 Session 004 — Dynamic clue display metadata

## Goal
Make AI Review clue cards feel more content-aware by deriving icon, tint, and subtitle copy from the clue label itself, while preserving the existing tag-based fallback and retrieval behavior.

## Files Changed
- `Life Narattor/Screens/SearchScreen.swift`
- `Docs/VERIFICATION_BACKLOG.md`
- `Docs/04_Sessions/2026-04-28_session-004.md`
- `Docs/05_Changes/Change-212-dynamic-review-clue-display.md`

## Implementation Notes
- Added `ReviewClueDisplay` as a small local presentation model for AI Review clue cards.
- The display model first matches user-visible tag names against a keyword map, then falls back to the existing tag type mapping.
- Added content-aware mappings for work/planning, morning starts, sleep/rest, emotion/stress, body/food/exercise, games/entertainment, learning/output, and relationships/communication.
- Card icons now use the inferred display metadata instead of only the tag type.
- Card subtitles now describe the likely meaning of the clue, for example `和计划、任务节奏有关` or `和一天开始后的状态有关`.
- Existing behavior is unchanged: clue cards still use the same `ReviewClueSuggestion`, selected tag filter, recent 90-day range, and immediate AI Review submission flow.
