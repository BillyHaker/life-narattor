## Goal
Replace the fragmented first-run guide with a clearer onboarding flow that has one product purpose: help users start recording lightly and understand how AI becomes useful later.

## Plan
1. Inspect existing onboarding copy and trigger path.
2. Reduce onboarding from four feature pages to three usage-path pages.
3. Remove multi-item checklist copy from each page.
4. Keep one concrete example per page.
5. Update Settings replay label from `4 步` to `3 步`.
6. Update verification backlog and project logs.
7. Verify build and tests.

## Detection Plan
- Expected behavior: first-run onboarding reads as a single path rather than a collection of feature tips.
- Pass criteria: three pages only; final CTA is `开始记一句`; skip CTA is `先进入看看`; Settings replay says `3 步`; build and tests pass.
- Failure signal: any onboarding copy still references `第四步`, `4 步`, or checklist-style `points`; visual hierarchy feels like scattered tips rather than a usage path.
- Regression surface: first launch after privacy consent, Settings replay, onboarding page navigation, root entry into Record tab.

## Implementation
- Rewrote `OnboardingGuideScreen` page data into three steps: low-pressure capture, direct-vs-assistant capture choice, and later AI review value.
- Replaced per-page bullet checklists with one `guideBlock` and one `exampleBlock`.
- Updated header and button copy to match the product promise.
- Updated Settings guide replay value to `3 步`.
- Updated verification backlog entry VRF-041.
