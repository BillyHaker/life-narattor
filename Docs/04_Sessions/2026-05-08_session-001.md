## Goal
Fix the Timeline range mismatch where the top story snapshot uses completed periods while the day list still reads the current live period.

## Plan
1. Align Timeline list ranges with snapshot ranges: yesterday, last completed 7 days, and last completed 30 days.
2. Tighten empty-state copy so the selected range is unambiguous.
3. Hide zero-value snapshot metric pills when there is no material.
4. Verify build and diff hygiene.
5. Record verification and rollback notes.

## Detection Plan
- Expected behavior: selecting `昨日` shows yesterday's snapshot and yesterday's day cards only.
- Expected behavior: selecting `7天回顾` or `30天回顾` excludes today and includes only completed days.
- Pass criteria: top snapshot counts and lower day cards refer to the same date range.
- Failure signal: snapshot says 0 material while visible day cards from a different period are shown.
- Regression surface: Timeline tab switching, empty states, snapshot refresh, day-detail navigation.

## Implementation
- Changed Timeline day-list date windows to completed periods:
  - `昨日`: yesterday start to today start.
  - `7天回顾`: seven days before today start to today start.
  - `30天回顾`: thirty days before today start to today start.
- Kept the existing fetch predicate `createdAt >= start AND createdAt < end`, so today is excluded from completed-period ranges.
- Updated yesterday empty-state copy to avoid implying today's records should appear in Timeline.
- Hid snapshot metric pills when both day count and material count are zero.
- Added ADR-018 to preserve the completed-period Timeline decision.
