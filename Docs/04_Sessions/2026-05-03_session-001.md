## Goal
Improve the production first-run experience by adding a concise onboarding guide, making the bottom navigation larger and less cramped on Pro Max devices, and keeping developer tooling out of the production root navigation.

## Plan
1. Replace the native root `TabView` with a custom three-item production shell.
2. Add a first-run onboarding guide that explains ordinary recording, assistant-assisted record creation, Timeline AI review, and AI Review.
3. Add a Settings entry to replay the onboarding guide.
4. Adjust the record input area spacing so it breathes above the larger bottom navigation.
5. Verify build, static diff hygiene, and preview rendering.
6. Record the UX decision, change log, and rollback notes.

## Implementation
- Added `app.hasSeenProductGuide` first-run state after privacy consent.
- Added `OnboardingGuideScreen` with three swipeable pages and skip/continue controls.
- Added a custom `RootShellView` and `RootTabBar` for the three production tabs: Record, Timeline, and AI Review.
- Made root tabs lazy-loaded so Timeline and AI Review are not instantiated until first selected.
- Added a Settings action to replay the onboarding guide.
- Added a small bottom padding adjustment to the record input area to fit the larger root bar.
