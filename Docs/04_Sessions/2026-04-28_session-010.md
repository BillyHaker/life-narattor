# 2026-04-28 Session 010 — Record all scope bottom-anchor scroll fix

## Goal
Make the Record page reliably land near the newest record when switching to `全部`.

## Context
The previous UUID-row scroll approach still left the view at the oldest record in manual testing. The likely cause is that `ScrollViewProxy.scrollTo(latestCaptureID)` was not consistently finding or laying out the row inside the lazy sectioned list at the moment the scope changed.

## Implementation Notes
- Added a stable invisible bottom anchor at the end of the record `LazyVStack`.
- Changed automatic latest scrolling to target the bottom anchor instead of a specific row UUID.
- Removed the unused `latestCaptureID` lookup.
- Kept the existing oldest-to-newest list order unchanged.
- Kept the delayed main-queue scroll so the filtered list can lay out before `scrollTo` runs.
- No data, search, filter, persistence, or capture creation logic changed.
