# 2026-05-12 Session 001 - Dismiss Keyboard After Record Send

## Goal
After the user sends a text record from the record page, the input method should automatically dismiss so the completed record can be seen without the keyboard staying open.

## Plan
1. Inspect the record page input bar and send flow.
2. Reuse the existing `dismissKeyboard()` focus release helper.
3. Route the bottom input bar send action through a small view-level wrapper.
4. Keep the change local to the record page to avoid touching persistence, AI, voice, or input component behavior.
5. Verify build and test suite.
6. Log the change and commit one focused patch.

## Work Log
- Confirmed `CaptureInputBarView` already accepts an external `FocusState` binding.
- Confirmed `RecordFeedScreen` already owns `isInputFieldFocused` and a `dismissKeyboard()` helper.
- Replaced the direct `viewModel.addCaptureFromInput` send action with `sendCurrentInput()`.
- `sendCurrentInput()` checks that the user had sendable text, calls the existing view model send flow, then dismisses the keyboard.
- Did not touch untracked `DesignMocks/` or `build/` folders.
