## Implementation Notes
- Confirmed the `Dev` tab was already excluded from Release/TestFlight by `#if DEBUG`.
- Added an explicit `FeatureFlags.isDeveloperMenuVisible` gate so Debug builds also hide `Dev` by default.
- Release builds always return `false` for the developer menu flag, even if a persisted UserDefaults value exists.
- Root navigation now only creates the `DevToolsRootView` tab in Debug and only when the internal flag is explicitly enabled.
- Updated beta checklist language to make the Release/Archive no-Dev visual check a must-complete item.
- Added the easy-to-miss App Store Connect items: build selection, pricing, category, privacy URL, support URL, privacy disclosure consistency, review notes, production backend health, usage quota behavior, and final smoke test.
