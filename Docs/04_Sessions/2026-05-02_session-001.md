# 2026-05-02 Session 001

## Goal
Reduce public App Store AI cost risk by adding backend usage tiers, quota overrides, and clearer admin usage visibility while keeping the iOS client API contract unchanged.

## Context
The app is now available through the App Store. Public users need constrained free AI usage, while trusted testers/reviewers and later paid users need higher limits. User-provided AI keys are a future option, but should not be exposed before quota and usage visibility are stable.

## Implementation
- Replaced the single backend quota table with tiered limits: `free`, `pro`, and `reviewer`.
- Made public users default to the `free` tier.
- Added environment-based tier controls:
  - `USAGE_DEFAULT_TIER`
  - `USAGE_PRO_USER_IDS`
  - `USAGE_REVIEWER_USER_IDS`
  - `USAGE_TIER_OVERRIDES`
  - `REVIEW_WHITELIST` as reviewer-compatible input
- Added `USAGE_LIMIT_OVERRIDES` for emergency quota tuning without code changes.
- Added `USAGE_STORE_PATH` so production usage data can be stored outside the system temp directory.
- Reduced default free daily limits for high-cost surfaces, especially assistant, review, and transcription.
- Preserved enough free internal processing quota for lightweight recording, cleaning, atomization, and tag suggestions.
- Extended usage events with tier, model/provider, and estimated input tokens.
- Updated the admin dashboard to show tier totals, estimated tokens, per-user tier, per-event model/provider, and estimated tokens.
- Documented the current managed-backend approach and reserved user-provided OpenAI-compatible API keys for a later settings surface.
