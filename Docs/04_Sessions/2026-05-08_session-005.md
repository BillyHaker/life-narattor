## Goal
Restore the preferred three-layer Record bottom UI while making the app-level tab bar larger and closer to full screen width.

## Plan
1. Restore the `记录 / 助手` segmented control above the input row.
2. Remove the inline Assistant button from the composer.
3. Keep the enlarged input row sizing from the previous pass.
4. Expand the root tab capsule horizontally and keep its larger hit target.
5. Verify build and document manual checks.
6. Commit one small follow-up change.

## Detection Plan
- Expected behavior: Record screen bottom appears as three layers: `记录 / 助手`, `麦克风 + 输入栏 + 发送`, and `记录 / 时间线 / AI 回顾`.
- Pass criteria: the old segmented `记录 / 助手` switch is visible again; the composer has no inline `助手` button; the app-level tab capsule spans closer to the screen width; record and assistant modes still switch correctly.
- Failure signals: input bar disappears, assistant cannot be selected, inline Assistant button still appears, or the root tab remains a small floating capsule.
- Regression surface: Record capture, Assistant session entry, voice recording state, keyboard layout, root tab switching.
