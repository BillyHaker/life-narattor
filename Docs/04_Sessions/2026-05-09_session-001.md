## Goal
Reduce over-assertive causal wording in record atomization without changing the atomization schema, storage model, or UI.

## Diagnosis
- The observed split result turned adjacent facts into definite causality: `因为昨天睡得晚，今天醒得嗓子疼`.
- The original record did not explicitly state causality, so the model inferred too much from adjacency.
- Existing atomization prompt used `caused by the matter`, which could over-encourage causal framing.

## Plan
1. Keep the existing atomization schema and write pipeline unchanged.
2. Tighten iOS direct OpenAI atomization prompt to require explicit causal markers before using definite causality.
3. Tighten backend `/v1/atomize` prompt with the same rule.
4. Add a small programmatic guard before app-side Core Data write.
5. Add unit tests for causal and non-causal examples.
6. Verify backend syntax and iOS build/test.

## Detection Plan
- Example A: `昨天睡得晚，今天醒来嗓子疼。` must not produce definite causal wording.
- Example B: `因为昨天睡得晚，所以今天很困。` may preserve definite causal wording.
- Example C: `最近压力很大，胃口也不好。` must not become `压力导致胃口不好`.
- Example D: `今天外界冷，火车上却挺热。` should remain a parallel/contrast fact.
