## Goal
Fix the Record screen bottom layout so the three preferred layers are all visible: `记录 / 助手`, the capture input row, and the app-level root tab bar.

## Plan
1. Confirm whether the input row disappeared or was covered by the root tab bar.
2. Keep the layered interaction model from ADR-021.
3. Add explicit bottom avoidance space for the root tab bar in the Record bottom inset.
4. Verify build and document manual checks.
5. Commit one small layout fix.

## Detection Plan
- Expected behavior: the Record bottom area displays `记录 / 助手`, then `麦克风 / 输入栏 / 发送`, then the wider app root tab bar.
- Pass criteria: the input row is fully visible above the root tab bar; recording chip is also visible when recording; root tab remains wide.
- Failure signals: input row is hidden behind the root tab bar, the `记录 / 助手` switch overlaps with the root tab, or the bottom controls float too high leaving excessive empty space.
- Regression surface: record capture, assistant mode switch, voice recording state, keyboard layout, root tab switching.

## Work Log
- Diagnosed the screenshot as a bottom safe-area stacking issue: the Record bottom inset was rendered behind the outer `RootTabBar`.
- Added an explicit `rootTabBarAvoidanceHeight` to `RecordFeedScreen` and applied it as bottom padding to the bottom inset.
- Kept the three-layer interaction and the wider root tab from the previous session.
