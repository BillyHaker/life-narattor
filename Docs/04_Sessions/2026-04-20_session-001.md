# Session 001

## Goal
- Let the record screen dismiss the keyboard when the user taps outside the search field or the bottom input field.

## Plan
- Add shared focus management for the search field and the bottom input field.
- Dismiss focus from the content areas only, so normal input taps are preserved.
- Keep list taps and detail opening behavior intact.

## Changes
- Added shared `FocusState` management in `RecordFeedScreen` for the search field and the bottom input field.
- Wired the record search field to the new focus state.
- Passed the input focus binding down into `CaptureInputBarView` so the parent screen can dismiss it reliably.
- Added background/content-area keyboard dismissal behavior to the record list, assist conversation scroll area, and empty-state area.
- Added a small focus helper in `CaptureInputBarView` so the input bar still works when used without an external focus binding.
