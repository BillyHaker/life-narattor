# Session Log

## Goal
把文本整理升级为混合链路：默认规则清理，复杂文本自动走 AI，且“重新整理”默认强制触发 AI。

## Plan
1. 扩展 `CleanDefiller`，增加复杂度判定。
2. 为 `AIService` 新增独立 `cleanTranscript(...)` 接口。
3. 接入 backend `/v1/clean` 与 OpenAI 直连 clean schema。
4. 将文本记录、语音转写、revision、详情页“重新整理”统一接到同一 clean 入口。
5. 验证 build 与 server 语法。

## Work Log
- 在 `CleanDefiller.swift` 中增加复杂度分析：重复、口头词密度、无标点长句、中英混杂、规则清理后仍脏。
- 在 `AIService.swift` 中新增 `cleanTranscript(text:forceAI:)`，分 Mock / OpenAI / Backend 三条实现。
- 在 `server/server.js` 中新增 `/v1/clean`，专门处理文本 clean，不复用聊天或归档接口。
- 在 `CaptureFeedViewModel.swift` 中新增统一入口：
  - `scheduleClean(captureID:sourceText:forceAI:)`
  - `resolveCleanResult(for:forceAI:)`
- 文本记录创建后、转写成功后、revision 写回后都会进入统一 clean 流程。
- 详情页“整理后”增加 `重新整理` 按钮，默认强制 AI clean。
- 手动重新整理时会清掉旧 atoms，并在新 clean 完成后重新进入拆分。
