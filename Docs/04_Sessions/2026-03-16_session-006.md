# Session Log

## Goal
为显性标签建议增加轻量元数据和排序规则，明确“哪些标签更适合出现在建议列表里”，同时避免为了实现这一点引入新的数据库迁移。

## Plan
1. 审查当前标签建议写入链，确认最合适的落点是 `AtomTagStore` 的建议分配阶段。
2. 新增代码侧标签推荐元数据，不改 Core Data schema。
3. 用匹配分、推荐度、稳定性、范围和用户确认痕迹共同排序已有显性标签建议。
4. 对新标签候选增加更保守的过滤与排序。
5. 验证构建。

## Work Log
- 新增 `TagRecommendationMetadata.swift`，定义轻量元数据：
  - `recommendability`
  - `scope`
  - `stability`
- 为预置显性标签种子提供元数据配置；未命中的标签按 `TagType` 给出默认元数据。
- 在 `AtomTagStore` 中为已有显性标签建议加入排序：
  - AI 匹配分
  - 推荐度
  - 稳定性
  - 范围（specific 优先于 broad）
  - 历史确认次数
- 为新标签候选加入过滤：
  - 过滤过泛词
  - 过滤推荐度低的候选
  - 按元数据和匹配分排序后最多保留 5 个隐性建议
- 保持“显性标签只来自用户主动输入或接受推荐”的边界不变。
