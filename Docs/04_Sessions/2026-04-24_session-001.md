# Session 001

## Goal
- Add system-derived signals to review analysis material without polluting the user-visible or hidden tag library.

## Plan
- Keep system facts separate from `TagEntity`.
- Derive system signals at retrieval time from existing `CaptureEntity` fields.
- Carry those signals through the shared `NarrativeBriefUnit` model so both overview review and focused evidence analysis can use them.
- Relax only overview review admission so time-range review can analyze structured records even when ordinary tag score is zero.

## Changes
- Added `SystemSignalKind` and `SystemSignal`.
- Added `systemSignals` to `NarrativeBriefUnit`.
- Derived date, week, month, weekday, day part, input type, source, and processing state signals in `MemoryIndexStore`.
- Updated overview retrieval admission so structured records in the selected time range can enter analysis even when tag score is zero.
- Added `system_signals` to direct OpenAI analysis payloads.
- Added `system_signals` to backend analysis request DTOs.
- Updated backend proxy analysis prompts to treat `system_signals` as factual context.
