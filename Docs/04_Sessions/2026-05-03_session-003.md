## Goal
Add an iCloud private sync foundation so user records can survive reinstall/upgrade and eventually sync across Apple devices without introducing a custom Life Narrator account database.

## Plan
1. Replace the local-only persistent container with `NSPersistentCloudKitContainer`.
2. Add CloudKit persistent-history and remote-change options.
3. Add the app entitlement file for iCloud + CloudKit.
4. Preserve local data safety by removing destructive persistent-store fallback behavior.
5. Mirror the backend runtime user id to iCloud KVS for reinstall continuity.
6. Update first-launch, Settings, public privacy page, store copy, and release checklist to describe iCloud private sync accurately.
7. Verify Debug build, Release simulator build, entitlements, Core Data smoke behavior, and diff hygiene.
8. Log manual signed-device validation items and commit.

## Implementation
- Switched the Core Data stack to `NSPersistentCloudKitContainer` and configured `iCloud.com.jintaoha.Life-Narattor` for non-memory stores.
- Enabled persistent history tracking and remote change notifications for CloudKit mirroring compatibility.
- Added CloudKit entitlements and wired them into Debug and Release build settings.
- Removed destructive persistent-store recovery. Store load failure now logs and asserts instead of deleting local data.
- Added default values to non-optional programmatic Core Data attributes so the model is safer for CloudKit mirroring.
- Mirrored `LifeNarrator.BetaUserID` to `NSUbiquitousKeyValueStore` to keep the AI quota identity stable after reinstall when iCloud is available.
- Updated onboarding, Settings, privacy policy, landing page, App Store submission copy, and preflight checklist to describe iCloud private sync and the original-audio caveat.
