- Replaced the inline assistant draft confirm card with a full-screen draft editor.
- Assistant draft editing now happens before commit-to-record; split and tag generation remain after final confirmation.
- Added editable draft body generation from `AssistArchivePayload` and commit helpers that persist the edited draft before writing the record.
- `重新整理` from the draft editor now dismisses the editor first, then regenerates the draft so the user sees a fresh AI version.

- Demoted `重新整理` and `继续对话` into lighter secondary actions with icons.
- Switched title/body fields to custom card styling instead of default bordered controls for a cleaner draft-page feel.

## Dev tag browser
- Added a DevTools tag browser that lists all tags, not just visible ones.
- Supports switching between explicit, hidden, and all tags.
- Groups tags by tag type and shows whether each tag is visible/common plus current link count.
## Hidden tag generation fix
- Diagnosed the empty hidden-tags Dev view as a generation/write issue rather than a DevTools filtering bug.
- Updated tag suggestion instructions so AI now returns 2-5 hidden suggestions by default instead of treating them as optional.
- Added `target_hidden_suggestions` to both direct and backend tag-suggestion policy payloads.
- Relaxed local hidden-tag filtering to keep more concrete hidden candidates while still filtering generic or sentence-like noise.
- Verified server syntax and app build after the change.
## Dev rerun tag suggestions for recent captures
- Added a DevTools action to rerun tag suggestions for the most recent 10 captures.
- The rerun path reuses existing atomization payloads and current atoms rather than re-atomizing records.
- Suggested tag links are cleared before rerunning so fresh suggestions replace older suggested links without touching confirmed tags.
- Added status feedback for progress and final processed/skipped/failed counts.
## Remove Mock AI and clarify hidden-tag root cause
- Removed the Mock AI feature flag, factory branch, and explicit MockAIService usages.
- Replaced the fallback path with `UnavailableAIService` so missing AI configuration now fails clearly instead of silently producing synthetic responses.
- Updated previews/debug surfaces to stop depending on mock AI.
- Rechecked hidden-tag behavior after removal: the biggest blocker was the mock chain returning empty suggestions; local hidden-tag filtering remains a secondary tuning concern rather than the primary cause.
## Dev rerun tag diagnostics
- Added per-record diagnostics to the recent-10 tag rerun flow.
- The Dev tag browser now shows, for each recent capture, whether it was updated, skipped, failed, or updated without hidden suggestions.
- The rerun summary now includes the total number of hidden suggestions returned, so the hidden-tag issue can be diagnosed without guessing.
## Tag rerun candidate fix and hidden-tag weighting
- Narrowed the Dev tag-rerun candidate set to recent `log` records so assistant chat messages no longer flood the recent-10 batch with irrelevant skips.
- Added a fallback tag-suggestion input path: when a record has no `atomization_payload` but does have an `assist_archive_card`, the rerun flow now derives a minimal `AtomizeResult` from the assistant archive record units.
- Aligned the tag-suggestion structured-output schema on both app and backend so `score` is required-but-nullable, which removes the strict-schema HTTP400 failure mode for tag suggestion reruns.
- Kept hidden-tag generation relatively permissive at ingest, but reduced hidden-tag influence at retrieval time: hidden tags still help recall, while repeated hidden tags are favored in `topHiddenTags` and one-off hidden tags without tag-hint support are downweighted in ranking.
- Verified server syntax and a full iOS build after the change.
## Hidden tag normalization mapping layer
- Fixed the Dev tag-rerun input again by narrowing the recent-10 batch to records that already have both usable tag-suggestion input and actual atoms, so the maintenance action no longer spends most of its budget on guaranteed skips.
- Improved assistant-archive fallback inputs for tag suggestion reruns by carrying assistant unit titles into `context_attributes`, key points into `result_or_state`, and lightweight derived `tag_hints`, so assistant-derived records produce better tag-suggestion payloads.
- Added a hidden-tag normalization pipeline that does not mutate original tags or tag links.
- Introduced hidden-tag clustering plus per-cluster synonym normalization through AI, then stored the resulting raw-to-canonical mapping in an `ArtifactEntity` payload rather than changing Core Data schema or rewriting existing tag relations.
- Wired the retrieval/index layer to consult the canonical mapping when reading hidden tags, so hidden-tag standardization affects recall and review without rewriting source data.
- Added a DevTools hidden-tag maintenance action that can organize hidden tags into broad buckets, normalize near-synonyms inside each bucket, and show the resulting canonical groups.
- Verified backend syntax and a full iOS build after these changes.

## Beta launch checklist and API guardrails
- Added a beta launch checklist covering feature readiness, stability checks, and test-user rollout boundaries.
- Documented a backend-proxy-based free trial model so test users can use the product without receiving provider API keys directly.
- Proposed per-user daily quotas split by request type (chat, review, transcription, archive/split retries) plus per-minute rate limits.
- Captured the minimum backend requirements for beta: user identity, usage counting, daily quota enforcement, and explicit over-limit messaging.

## Identity, privacy, API, and export design
- Added a formal design draft covering stable user identity, beta-to-production inheritance, and the role of Apple ID as an auxiliary auth/recovery method.
- Documented a local-first privacy stance: record content stays on-device by default, while the backend stores identity, usage, and anonymous aggregate metrics.
- Defined the hosted-vs-user-supplied AI provider strategy and explicitly required strong protection for user-supplied API credentials.
- Documented long-term extract/export principles built around structured intermediate artifacts rather than UI-only views.

## Beta user identity and usage quota skeleton
- Added a first-pass backend identity/quota skeleton for beta users.
- Backend now expects a stable `X-User-Id`, records lightweight beta-user presence, and applies per-request-type daily quotas.
- Added lightweight file-backed usage accounting for beta testing, including request events and over-quota responses.
- iOS backend requests now attach a locally persisted beta user identifier, and transcription requests also send estimated audio duration for quota accounting.
