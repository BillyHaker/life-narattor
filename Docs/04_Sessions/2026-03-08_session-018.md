## Work Log (chronological)
1. Investigated `CaptureFeedViewModel` transcription queue and found duplicate enqueue race:
   - `stopRecording` called `loadCaptures()` first, and `loadCaptures` can auto-restore pending transcriptions.
   - Then `stopRecording` explicitly called `enqueueTranscription(...)` again.
2. Same duplicate pattern existed in `retryTranscription(...)`.
3. Applied minimal fix:
   - In both methods, call `enqueueTranscription(...)` before `loadCaptures()`.
   - This ensures restore logic sees an active task and does not enqueue a second competing task.
4. Rebuilt with `xcodebuild` and verified `EXIT:0`.
