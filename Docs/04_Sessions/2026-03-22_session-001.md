## Scope
- Add a lightweight beta admin dashboard.

- Expose usage summaries suitable for beta operations.

## Work Completed
- Added admin HTML routes at `/admin`, `/admin/users`, `/admin/users/:id`, and `/admin/invites`.
- Added lightweight admin access control via `ADMIN_TOKEN` when configured.
- Added invite persistence in `server/invite_store.js` with states for created, sent, used, revoked, and send failures.
- Added `POST /admin/invites` to generate and send invite codes and `POST /admin/invites/resend` to resend existing invites.
- Added `POST /v1/beta/register` so invite codes can mint stable beta `user_id` identities.
- Expanded beta-user storage helpers to support listing users and invite-based registration, including optional Apple subject binding.
- Expanded usage storage helpers so the admin dashboard can render daily totals, per-user summaries, recent events, and quota-hit visibility.
- Added Resend-based email sending support for invites when `RESEND_API_KEY` and `INVITE_EMAIL_FROM` are configured; otherwise invites still generate and record a clear unsent state.
- Verified the admin dashboard and beta registration flow with a localhost smoke test.
