# Session 001

## Goal
- Make the day detail `AI 的回应` empty state explain the real cause when records exist but AI review material is not ready.

## Plan
- Keep the existing AI review retrieval pipeline unchanged.
- Add per-capture processing state into the day detail local capture rows.
- Replace the single generic empty message with state-aware messages for no records, pending processing, split failure, and missing structured material.

## Changes
- Added `processingState` to the day detail `CaptureRow` model.
- Captured each day's Core Data `CaptureEntity.processingState` while loading day records.
- Replaced the generic `今天的记录还不够形成一段可信的 AI 回应。` message with state-aware copy:
  - no records
  - records saved but still being organized/split
  - all records split failed
  - partial split failure
  - records exist but no review-ready structured units are available
- Kept existing AI request behavior unchanged when `NarrativeMaterial.representativeUnits` exists.
