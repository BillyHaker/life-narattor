# Session Log

## Goal
为 `重新拆分` 和自动拆分过程补充明确的进行中状态提示，避免用户只看到模糊的“正在拆分…”。

## Plan
1. 为拆分链路增加轻量进度状态消息。
2. 通过现有通知机制把状态消息推到详情页。
3. 在详情页 `拆分` tab 中显示具体阶段文案。
4. 验证语法检查和 Xcode build。

## Work Log
- 在 `AtomizationCoordinator.swift` 中为拆分流程增加阶段性进度回调：
  - `正在加载标签库…`
  - `已发送拆分请求，等待 AI 响应…`
  - `AI 已返回，正在整理拆分结果…`
  - `正在生成标签建议…`
- 在 `CaptureFeedViewModel.swift` 中扩展 `captureProcessingStateChanged` 通知，允许携带 `atomizationStatusMessage`。
- 手动 `重新拆分` 时，先推送 `已发送拆分请求…`，后续由拆分协调器继续更新具体阶段。
- 在 `CaptureDetailSheet.swift` 中新增本地状态 `currentAtomizationStatusMessage`，并在 `拆分` tab 中优先显示该文案。
- 拆分结束后，如果状态不再是 `splitting`，会自动清掉进行中提示，避免残留旧状态。
