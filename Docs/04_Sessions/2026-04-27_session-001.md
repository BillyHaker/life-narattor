# Session 001 — Timeline Range Label Alignment

## Goal
Align Timeline range labels with the actual rolling review logic so users do not read `本周 / 本月` as natural calendar week/month when the product is using rolling 7-day and 30-day review windows.

## Plan
- Rename Timeline tabs from natural-period language to rolling review language.
- Remove the duplicate `近30天` tab from the visible segmented control.
- Align Timeline list fetch windows with the displayed rolling ranges.
- Update empty-state and header copy to avoid `本周 / 本月` wording.
- Verify with build and unit tests.

## Detection Plan
- Expected behavior:
  - Timeline segmented control shows `今天 / 7天回顾 / 30天回顾`.
  - Timeline copy does not contain `本周` or `本月` in this screen.
  - Selecting `7天回顾` fetches the recent 7-day window, not the natural calendar week.
  - Selecting `30天回顾` fetches the recent 30-day window, not the natural calendar month.
- Detection path:
  - search Timeline files for `本周` and `本月`
  - open Timeline in simulator and switch tabs
  - inspect cards around week/month boundaries if possible
- Pass criteria:
  - no `本周 / 本月` labels remain in Timeline visible copy
  - build succeeds
  - unit tests pass
- Failure signals:
  - tab says rolling range while list still uses natural week/month
  - duplicate `30天回顾` and `近30天` tabs appear together
  - empty state still says `本周` or `本月`

## Work Log
- Added `TimelineScope.timelineTabs` so the visible Timeline segmented control only includes `今天`、`7天回顾`、`30天回顾`.
- Renamed `.week` title to `7天回顾` and `.month` title to `30天回顾`.
- Updated Timeline header and empty-state copy to use rolling-range language.
- Changed `.week` fetch interval from natural `weekOfYear` to the previous 7 days.
- Changed `.month` fetch interval from natural calendar month to the previous 30 days.
- Confirmed no `本周 / 本月` text remains in Timeline scope files.
