# AI 回顾固定评估样例

Last updated: 2026-04-27
Status: Evaluation fixture for beta regression

## 目标
AI 回顾不能只像普通搜索一样找关键词。它需要做到：
- 展示事实：明确引用记录里真实出现过的内容。
- 找到联系：指出线性日记里不明显的关系、变化和重复模式。
- 允许继续交流：给出自然的后续问题，用户点下去能继续追问。
- 保持轻量：不要长报告，不要像病历或客服纪要。

## 样本数据建议

### 样本 A：上班族，7 天内 8-12 条
1. 周一早上通勤很拖，路上一直在想工作，到公司前已经很累。
2. 周一下午新主管提到项目节奏偏慢，我有点紧张。
3. 周二早起后先处理最重要的任务，上午状态明显好一些。
4. 周三晚上胃口不好，晚饭吃得很少，还在想白天的评审。
5. 周四开会前很焦虑，但会后发现问题没有想象中严重。
6. 周五下班后去散步，心情比前几天松了一点。
7. 周六上午补觉，下午整理了下周要做的三个任务。
8. 周日晚上想到上班有点抗拒，但比上周日更清楚要先做什么。

### 样本 B：大学生，7 天内 8-12 条
1. 周一早八起不来，课上注意力很散。
2. 周二图书馆复习了两小时，发现自己不是不会，只是拖太久了。
3. 周三和室友聊到考研，突然有点焦虑。
4. 周四晚上打球以后状态变好，回宿舍还背了单词。
5. 周五小组作业推进很慢，我有点烦。
6. 周六上午刷剧太久，下午补了实验报告。
7. 周日晚上把下周课表和复习计划写了一遍。
8. 这一周最明显的是运动之后更愿意学习。

### 样本 C：兴趣/游戏，30 天内 10-15 条
1. 计划假期打游戏段位，但第一周一直没有上去。
2. 多次提到云顶之弈，重点在阵容理解和上分失败。
3. 有几次因为失败感到烦闷，但仍然在复盘策略。
4. 假期计划和工作/学习安排冲突，娱乐没有真正带来放松。
5. 记录里出现“人家是饼家，他们不一样的”这类零碎表达，需要保留原语境，不要强行解释。

## 固定问题
1. 我过去一周主要发生了什么变化？
2. 最近胃口不好，和工作压力有没有关系？
3. 我最近焦虑主要来自哪里？
4. 哪些事情反复出现，但我可能没有注意到？
5. 过去 30 天里，游戏和心情有什么关系？
6. 最近有没有一个值得继续观察的线索？
7. 这一段里有哪些事实，哪些只是推测？
8. 如果我要继续问，最值得问什么？

## 通过标准

### 内容质量
- 回答必须先给事实，再给联系，不直接跳到建议。
- 至少引用 2 条来自记录的具体事实，例如时间、行为、感受、场景。
- 联系必须使用谨慎语言，例如“可能有关”“看起来有对比”，不能把弱证据说成定论。
- 不使用“用户”“助手”“系统”“本次对话中”等产品或报告化语言。
- 输出语言跟记录一致，中文记录应输出中文。

### 交互质量
- 首轮回答应能在首屏内快速读完。
- 证据区默认可折叠或低干扰展示。
- 后续追问应像聊天一样接在下面，不把首轮分析挤成一大块。
- 推荐追问被点击后应自动开始追问，不只是填入输入框。

### 失败信号
- 只列关键词，没有解释关系。
- 编造记录里没有出现的事实。
- 把“没有足够材料”用于已有明确记录的周期。
- 输出英文或第三人称报告腔。
- 把隐性标签当作用户可见结论直接展示。

## 检测步骤
1. 用 Debug synthetic records 或手动输入创建样本 A/B/C。
2. 确认记录完成拆分，并在 Debug/Dev 中能看到 review material。
3. 打开 AI 回顾，逐条输入固定问题。
4. 记录每个问题的首轮输出是否满足内容质量和交互质量。
5. 点 2-3 个推荐追问，确认自动追问并保持聊天式阅读。
6. 打开时间线 `7天回顾` 和 `30天回顾`，确认故事线材料数量不为 0，且文案与周期一致。
