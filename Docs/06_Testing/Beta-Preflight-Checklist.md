# 测试版提审前准备工作清单

Last updated: 2026-03-22
Status: Current preflight checklist

## 目标
这份清单不是功能愿望单，而是面向“尽快把测试版本交给审核”的最后收尾清单。
它基于现有的 [Beta-Launch-Checklist.md](/Users/billyha/Desktop/Life%20Narattor/Docs/06_Testing/Beta-Launch-Checklist.md) 做收敛，只保留：
- 提审前必须完成或明确确认的事项
- 可以延后到测试用户阶段再做的事项
- 本版先隐藏或不上线的能力

---

## 一、提审前必须完成

### A. 产品边界确认
- [x] `AI 回顾` 作为回顾 Tab 的唯一主入口，不再暴露旧的普通搜索心智。
- [x] 周/月自动回顾入口继续隐藏，不进入当前测试版主路径。
- [x] Dev 专用能力不对测试用户开放。
- [x] 助手整理为记录采用“草稿预览 -> 轻编辑 -> 确认入库”的最终流程，不回退到小确认卡。

### B. 核心用户路径必须可用
- [ ] 文本记录创建、查看、编辑正常。
- [ ] 语音记录创建、转写、整理文本查看正常。
- [ ] 助手文本对话正常。
- [ ] 助手语音输入正常，转写后能发回当前助手线程。
- [ ] 助手“整理为记录”可打开全屏草稿页，确认后才入库。
- [ ] 记录详情页：原始 / 整理后 / 拆分 三个 tab 基本可用。
- [ ] 重新转写 / 重新整理 / 重新拆分 各自行为正确，不串链路。
- [ ] `AI 回顾`：示例问题点击后自动开始分析。
- [ ] `AI 回顾`：首轮输出为 `事实 / 联系 / 可继续问`，不是长报告。
- [ ] `AI 回顾`：继续追问以聊天气泡展示，不造成首屏混乱。

### C. 质量和稳定性
- [ ] 主工程 `xcodebuild` 通过。
- [ ] backend proxy 正常，关键 AI 路由无明显 400/500 常见错误。
- [ ] 没有明显的假成功：AI 未配置时明确失败，而不是返回假结果。
- [ ] 关键路径不出现明显卡死、重复请求或状态残留。
- [ ] launchd backend 启动稳定，admin 与 AI proxy 可以重新安装后恢复。

### D. 隐私与测试版说明
- [ ] 开局隐私声明文案已确定：默认记录内容只保存在本地。
- [ ] 不向测试用户暴露 provider API key。
- [ ] AI 请求全部经 backend proxy。
- [ ] 测试版已知边界说明已准备好（最少一版文字稿即可）。

## 二、建议在提审前完成，但可晚于审核前最后一刻

### A. 标签系统收尾
- [ ] 推荐标签显示数量控制在轻量范围内。
- [ ] 新推荐标签被接受时有轻反馈。
- [ ] 隐性标签标准化 mapping 至少跑过一轮，确认不会明显带偏 AI 回顾。
- [ ] Dev 中可查看显性/隐性标签，便于内部验收。

### B. 后台与测试运营准备
- [ ] admin 后台可查看用户、usage、邀请码。
- [ ] 邀请码生成和注册链路可用。
- [ ] 测试用户配额已启用。
- [ ] 邮件发送是否启用已明确：
  - 若启用，实测一封邀请邮件
  - 若暂不启用，后台仍能生成邀请码并记录

### C. 回归样本
- [ ] 至少用一批文本记录验证拆分与标签。
- [ ] 至少用一批语音记录验证转写、整理、拆分。
- [ ] 至少用一组 AI 回顾 overview 问题验证输出风格。
- [ ] 至少用一组 focused 问题验证证据与追问。

## 三、本版先隐藏 / 延后

- [ ] 周/月自动回顾正式入口
- [ ] 复杂长周期 narrative 生成入口
- [ ] 普通搜索页独立入口（由记录页内搜索替代）
- [ ] 测试用户邮件自动邀请的正式运营流程（可在真正发测试版前再接通）
- [ ] 用户自带 AI API 模式
- [ ] 完整云同步 / 内容恢复
- [ ] 高级 admin 操作（调额度、封禁、分组）

## 四、提审前手动验收顺序

### 1. 记录链
1. 创建文本记录
2. 创建语音记录
3. 打开详情页
4. 验证重新整理 / 重新拆分 / 重新转写

### 2. 助手链
1. 文本对话
2. 语音对话
3. 整理为记录
4. 草稿轻编辑
5. 确认入库

### 3. AI 回顾链
1. 点击示例问题
2. 查看 `事实 / 联系 / 可继续问`
3. 展开证据
4. 连续追问 2-3 轮

### 4. 标签链
1. 看推荐标签
2. 接受一个已有显性标签推荐
3. 接受一个新建议标签
4. Dev 中检查隐性标签与标准化结果

### 5. 后台链
1. 打开 admin
2. 查看 usage 总览
3. 生成邀请码
4. 验证注册入口

## 五、上线前决定项

这些不是代码任务，但在提审前需要明确：
- [ ] 当前测试版是否允许真实外部测试用户进入，还是先内部 TestFlight。
- [ ] 是否本轮就开启邀请码邮件发送。
- [ ] 是否给测试用户开放语音功能，还是先内部使用。
- [ ] 是否在测试版中显示“本地优先存储”的隐私说明。
- [ ] 哪些 Dev 能力必须确保不出现在测试用户可见界面。

## 六、完成标准

这份 preflight 清单可以视为完成，当且仅当：
- `必须完成` 部分全部打勾
- `建议完成` 部分至少完成大部分关键项
- `本版先隐藏 / 延后` 已有明确处理，不会误暴露给测试用户
- build 通过
- 核心路径已按顺序手测一轮
