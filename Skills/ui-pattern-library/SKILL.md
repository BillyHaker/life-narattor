# UI Pattern Library (Meta)

## Input patterns
- Chat-like input + feed
- Card composer
- Inbox quick capture
- Timeline-first capture

## Browse patterns
- Day grouped list
- Timeline sections
- Project/tag filtered list

## Review patterns
- Daily narrative
- Weekly/monthly digest
- Project/storyline review

## Usage rule
Always propose A/B/C with best-for + tradeoff + complexity (low/med/high).

## Navigation patterns

Use these patterns to structure how users move between major sections of your app.
- Bottom tab bar (5 or fewer primary sections)
- Side drawer / hamburger menu (many sections or secondary navigation)
- Top segmented control or tabs (quick context switching)
- Floating action menu (contextual actions accessible anywhere)

## Modal & overlay patterns

Choose appropriate overlays to capture user focus without losing context.
- Full‑screen modal (critical tasks that require attention and may block navigation)
- Bottom sheet / slide‑up panel (lightweight actions that complement the current screen)
- Popover or tooltip (small context or help)
