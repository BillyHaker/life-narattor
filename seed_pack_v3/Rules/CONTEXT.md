# CONTEXT (Universal)

To keep tasks focused and avoid unintended side effects, follow these context management guidelines:

- **Read only what you need** – Only open the files relevant to the current task.  Avoid scanning or modifying unrelated parts of the codebase.
- **Keep changes minimal and scoped** – Make small, incremental changes that are clearly tied to the goal.  Avoid opportunistic refactors or stylistic cleanups outside of the task’s scope.
- **Ask before structural changes** – Before performing large structural changes (e.g. moving multiple files, deleting modules), create an ADR and seek confirmation through a handover.
- **Clarify ambiguous scope** – If the task description is unclear or seems too broad, ask clarifying questions or propose a scoped plan before proceeding.
